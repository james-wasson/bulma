Grid items are implicit meaning any item under a grid becomes a grid child.

{% include elements/anchor.html name="Control" %}

You can specify a childs position using one of two modfiers:

  • is-left-* e.g. is-left-1
  • is-top-* e.g. is-top-3

By default Bulma supports positioning counts up to 12. You can change this default by setting $grid-column-row-count to a larger value.

You can use the below modifers to specify the side to position against:

  • is-left-inverse - Offset against the right side of the grid
  • is-top-inverse - Offset against the bottom side of the grid

If you want to use the modifer is-top-inverse you mut specify a row count via has-row-count-* on the grid

  • is-width-* e.g. is-width-1
  • is-height-* e.g. is-height-3

{% include elements/anchor.html name="Responsiveness" %}

Responsive modifiers can be combined with the following grid child modfiers:

  • is-left-*
  • is-top-*
  • is-width-*
  • is-height-*

Take is-widthfor example:

  • is-width-*-mobile
  • is-width-*-tablet
  • is-width-*-touch
  • is-width-*-desktop
  • is-width-*-widescreen
  • is-width-*-fullhd
{% include elements/anchor.html name="Flow" %}

You can change the flow of children through the grid using one of two modfiers:

  • has-flow-column - Fills the first column then the second and so on
  • has-flow-dense - Moves others to fill the gap left behind by a positioned child

By default the grid flows via row.
